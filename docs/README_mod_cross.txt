杂交计划模块 (mod_cross.R) 使用指南
===============================================================================

📁 文件位置：
  - 核心模块：R/mod_cross.R
  - 使用示例：scripts/example_cross_plan.R

===============================================================================
🚀 快速开始
===============================================================================

1. 加载模块：
   source("R/mod_cross.R")

2. 创建杂交计划（使用亲本名称）：
   result <- create_cross_plan(
     batch_name = "2025春季批次",
     mothers = c("中黄301", "南农66", "皖宿112"),
     fathers = c("天辰6号", "油6019", "南农47"),
     include_reciprocal = TRUE,
     use_id = FALSE
   )

3. 创建杂交计划（使用亲本ID，更高效）：
   result <- create_cross_plan(
     batch_name = "2025春季批次",
     mothers = c("P0001", "P0007", "P0014"),
     fathers = c("P0101", "P0102", "P0103"),
     include_reciprocal = TRUE,
     use_id = TRUE
   )

===============================================================================
📋 核心功能
===============================================================================

1. create_cross_plan() - 创建杂交计划
   功能：根据母本和父本列表创建杂交计划
   特点：
   - 自动检查已存在的组合，避免重复
   - 自动过滤自交（母本=父本）
   - 可选是否包含反交
   - 批量插入数据库
   - 返回详细的计划摘要

   参数：
   - batch_name: 批次名称（必填）
   - mothers: 母本列表（必填）
   - fathers: 父本列表（必填）
   - db_path: 数据库路径（默认："data/db/soy_cross.db"）
   - include_reciprocal: 是否包含反交（默认：TRUE）
   - status: 初始状态（默认："planned"）
   - use_id: 是否使用ID而非名称（默认：FALSE）

   返回值：
   - summary: 计划摘要统计
   - new_crosses: 新创建的组合
   - skipped: 跳过的已存在组合
   - db_updated: 数据库是否已更新

2. has_cross() - 检查组合是否存在
   功能：检查特定母本-父本组合是否已存在
   
   示例：
   exists <- has_cross("中黄301", "天辰6号", use_id = FALSE)
   exists <- has_cross("P0001", "P0101", use_id = TRUE)

3. summarize_cross_batches_db() - 查询批次统计
   功能：统计数据库中各批次的杂交组合数量
   
   示例：
   # 查询所有批次
   all_batches <- summarize_cross_batches_db()
   
   # 查询特定批次
   specific <- summarize_cross_batches_db(
     batch_filter = c("2025春季批次", "2025夏季批次")
   )

===============================================================================
💡 使用技巧
===============================================================================

1. 使用ID vs 使用名称：
   - 使用ID（use_id = TRUE）更高效，推荐大批量操作时使用
   - 使用名称（use_id = FALSE）更直观，适合小规模操作

2. 批次命名建议：
   - 使用有意义的名称，如"2025春季批次"、"G25b1"
   - 便于后续筛选和管理

3. 自动去重：
   - 系统会自动检查已存在的组合
   - 跳过重复组合，返回在 result$skipped 中

4. 反交处理：
   - 默认自动创建反交（A×B 和 B×A）
   - 不需要反交时设置 include_reciprocal = FALSE

5. 批量操作：
   - 可以从数据库查询符合条件的亲本
   - 一次性创建大量组合

===============================================================================
📝 工作流程示例
===============================================================================

步骤1：加载模块
source("R/mod_cross.R")

步骤2：准备亲本列表（方式1：手动指定）
mothers <- c("中黄301", "南农66", "皖宿112")
fathers <- c("天辰6号", "油6019")

步骤3：准备亲本列表（方式2：从数据库查询）
library(RSQLite)
con <- dbConnect(SQLite(), "data/db/soy_cross.db")
mothers <- dbGetQuery(con, "SELECT name FROM parents WHERE 转基因 = '是'")$name
fathers <- dbGetQuery(con, "SELECT name FROM parents WHERE 转基因 = '否'")$name
dbDisconnect(con)

步骤4：创建杂交计划
result <- create_cross_plan(
  batch_name = "2025春季批次",
  mothers = mothers,
  fathers = fathers
)

步骤5：查看结果
print(result$summary)         # 查看摘要
print(result$new_crosses)     # 查看新建组合
print(result$skipped)         # 查看跳过的组合

步骤6：验证结果
batches <- summarize_cross_batches_db()
print(batches)

===============================================================================
⚠️ 注意事项
===============================================================================

1. 数据库路径：
   - 默认使用 "data/db/soy_cross.db"
   - 确保数据库文件存在且包含 crosses 和 parents 表

2. 亲本必须存在：
   - 使用名称时，亲本必须在 parents 表中存在
   - 不存在的亲本会被跳过并给出警告

3. 自交过滤：
   - 系统自动过滤母本=父本的情况
   - 无需手动处理

4. 事务安全：
   - 使用数据库事务确保数据一致性
   - 插入失败会自动回滚
