# 杂交组合分析模块开发框架 (mod_analysis.R)

## 1. 模块概述

**目标**：提供一套工具函数，用于从数据库中提取、统计和可视化杂交组合的历史数据，帮助育种家了解亲本利用情况、评估组合实施进度及质量。

**文件路径**：`R/mod_analysis.R`
**依赖**：`DBI`, `RSQLite`, `dplyr`, `tidyr` (可选), `ggplot2` (可选)

## 2. 核心功能设计

### 2.1 基础统计 (Basic Statistics)

提供对杂交数据库的概览性统计。

* **`summarize_batches_db(db_path)`**

  * **功能**：按批次统计组合数量、不同状态（planned, success, failed）的比例。
  * **SQL 逻辑**：`SELECT batch, status, COUNT(*) FROM crosses GROUP BY batch, status`
  * **用途**：快速查看各季度的任务量和完成度。
* **`calculate_success_rate_db(db_path, by_batch = TRUE)`**

  * **功能**：计算总体或各批次的杂交成功率。
  * **逻辑**：成功数 / (成功数 + 失败数)。
  * **用途**：评估技术水平或环境影响。

### 2.2 亲本利用分析 (Parental Usage Analysis)

分析哪些亲本是“核心亲本”，哪些是“冷门亲本”。

* **`get_parent_usage_stats(db_path, top_n = 20)`**

  * **功能**：统计每个亲本作为母本、父本及总计的使用频次。
  * **输出**：数据框 `(parent_id, name, female_count, male_count, total_count)`。
  * **用途**：
    * 识别过度使用的亲本（可能导致遗传狭窄）。
    * 识别未充分利用的优异资源。
* **`analyze_parent_combinations(db_path, parent_id)`**

  * **功能**：查询指定亲本的所有配组对象。
  * **用途**：查看某个核心亲本已经和哪些材料配过，避免重复配组。

### 2.3 组合与正反交分析 (Combination & Reciprocal Analysis)

* **`check_reciprocal_pairs(db_path)`**

  * **功能**：检查哪些组合已经做了正交，是否缺反交。
  * **逻辑**：对比 `(A, B)` 和 `(B, A)` 的存在情况。
  * **输出**：列表，包含“仅正交”、“仅反交”、“双向均有”的组合名单。
* **`get_cross_matrix_data(db_path, batch = NULL)`**

  * **功能**：生成用于绘制热力图的矩阵数据。
  * **格式**：长格式 `(female_id, male_id, value)`，value 可以是次数、状态或粒数。
  * **用途**：直观展示亲本间的配组覆盖度（稀疏矩阵可视化）。

## 4. 开发计划

1. **第一阶段**：实现 `summarize_batches_db` 和 `get_parent_usage_stats`，满足最基本的“做了多少、用了谁”的问题。
2. **第二阶段**：实现 `check_reciprocal_pairs`，辅助查漏补缺。
3. **第三阶段**：结合可视化库（如 ggplot2 或 Highcharts），提供直观图表。

## 5. 使用场景示例

**场景：在此次春季配组前，我想看看“中黄13”过去都和谁配过，还没和谁配？**

```r
# 1. 加载模块
source("R/mod_analysis.R")

# 2. 查看中黄13的历史配组
zh13_history <- analyze_parent_combinations(db_path, parent_name = "中黄13")
print(zh13_history) 
# 输出：
# 2023夏: x 冀豆12 (成功)
# 2024春: x 齐黄34 (失败)

# 3. 查看高频使用的父本，避免再次重复使用
top_fathers <- get_parent_usage_stats(db_path) %>% arrange(desc(usage_male))
print(head(top_fathers))
```
